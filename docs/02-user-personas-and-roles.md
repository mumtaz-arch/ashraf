# User Personas and Roles

## 1. Guest / Visitor
### Description
A public visitor who wants to book a photography service.

### Goals
- understand services quickly
- check available options
- submit a booking easily
- know what happens after booking

### Permissions
- view landing page
- view services/packages
- access booking form
- submit booking request
- optionally check booking reference later (future)

---

## 2. Registered Customer (Future-Ready)
### Description
A customer account holder for future phases.

### Goals
- track bookings
- manage profile
- see booking history
- reschedule requests (future)

### Permissions (Future)
- login
- manage profile
- view booking history
- request reschedule
- upload payment proof (future)

---

## 3. Admin / Photographer
### Description
The main operator managing appointments and schedule.

### Goals
- see all incoming bookings
- confirm/reject/reschedule bookings
- manage available dates and time slots
- prevent conflicts
- keep workflow simple

### Permissions
- login to admin area
- view booking list
- view booking details
- update booking status
- manage services/packages
- manage availability
- block unavailable slots
- mark booking completed/cancelled

---

## 4. Staff (Optional Future Role)
### Description
Assistant who helps manage bookings.

### Future Permissions
- limited booking management
- limited schedule management
- no system-level settings
